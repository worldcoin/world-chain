## Abstract

**World ID Accounts** are accounts whose address is derived from a World ID v4 OPRF nullifier. A new EIP-2718 transaction type (`0x6f`) authenticates these accounts via signatures from keys delegated through a World ID uniqueness proof. The proof's `signal` commits to the authorized key set. Accounts are ephemeral — they can be destroyed and recreated at a new address, preserving only a monotonic nonce.

`leaf_index`, account address, WorldIDRegistry authenticator, and delegated signing keys are all unlinkable. The monotonic nonce gives RPs access to a chain-wide history of a given World ID, over a global `rpId = 480 (World Chain RP)` enabling expressive verification over historical actions.

## Motivation

1. **Multi-dimensional gas fees** — All `0x6f` transactions originate from verified humans by construction. The address derivation IS the proof.
2. **Sybil-resistant primitive** — One human, one address per chain. Rotatable. The abstract authenticator registry decoupled from the tree makes World Chain an address book by default.
3. **Native multi-auth** — Passkey (P256/WebAuthn) support with private multi-auth for World ID holders.

### Account Model

A World ID Account is an address derived from a proven **account nullifier**. A precompile at `WORLD_ID_ACCOUNT_REGISTRY` manages global nullifier tracking over `rp_id = 480`.

| Nullifier | Purpose | Action Derivation |
|-----------|---------|-------------------|
| **Creation** | track account recycle count | `keccak256("WORLD_ID_ACCOUNT_CREATION")` |
| **Account** | pub key derivation | `keccak256(WORLD_CHAIN_RP_ID \|\| world_id_nonce \|\| generation)` |


### WorldIDAccountFactory — Precompile

Deployed at `address(0xc0ffee)`. Manages account creation and recreation via proof verification.

```solidity
/// @notice Pseudo-code — actual implementation is a native precompile.
contract WorldIDAccountFactory {

    // ─── State ─────────────────────────────────────────────────────────
    mapping(uint256 => uint256) public accountCreations;       // creation_nullifier → generation
    mapping(uint256 => bool)    public spentAccountNullifiers;  // account_nullifier → spent
    mapping(address => bool)    public destroyed;               // account → destroyed flag

    // ─── Types ─────────────────────────────────────────────────────────
    enum AuthType { Secp256k1, P256, WebAuthn }

    struct AuthorizedKey {
        AuthType authType;
        bytes    keyData;    // 33 bytes (secp256k1) or 64 bytes (P256/WebAuthn)
    }

    // ─── Events ────────────────────────────────────────────────────────
    event AccountCreated(address indexed account, uint256 generation);
    event AccountRecycled(
        address indexed oldAccount,
        address indexed newAccount,
        uint256 newGeneration
    );
}
```

#### `createAccount`

Creates a new World ID Account. Requires **two** `OPRFNullifierProof`s.

```solidity
function createAccount(
    uint256[5] calldata creationProof,
    uint256[5] calldata accountProof,
    AuthorizedKey[] calldata keys
) external returns (address account) {

    // ① Verify creation proof
    //   action  = keccak256("WORLD_ID_ACCOUNT_CREATION")
    //   signal  = 0
    //   → proves leaf_index has POH credential
    //   → extracts creation_nullifier
    uint256 creationNullifier = _verifyCreationProof(creationProof);

    // ② Read generation count
    uint256 generation = accountCreations[creationNullifier];

    // ③ Verify account proof
    //   action  = keccak256(abi.encodePacked(WORLD_CHAIN_RP_ID, uint256(0), generation))
    //   signal  = keccak256(abi.encode(keys)) >> 8
    //   → proves same leaf_index, signal commits to key set
    //   → extracts account_nullifier
    uint256 accountNullifier = _verifyAccountProof(generation, keys, accountProof);

    // ④ Enforce nullifier uniqueness
    require(!spentAccountNullifiers[accountNullifier]);
    spentAccountNullifiers[accountNullifier] = true;

    // ⑤ Derive address
    account = address(bytes20(keccak256(abi.encodePacked(accountNullifier))));
    require(!destroyed[account]);

    // ⑥ Initialize account
    //   → write keys to AUTHORIZED_KEYS_SLOT
    //   → write generation to GENERATION_SLOT
    //   → set EIP-7702 delegation to WORLD_ID_ACCOUNT_DELEGATE

    // ⑦ Increment generation
    accountCreations[creationNullifier] = generation + 1;

    emit AccountCreated(account, generation);
}
```

#### `recreateAccount`

Atomically destroys old account and creates new one. Requires **three** `OPRFNullifierProof`s.

```solidity
function recreateAccount(
    uint256[5] calldata creationProof,
    uint256[5] calldata destructionProof,
    uint256            oldAccountNullifier,
    uint256[5] calldata accountProof,
    AuthorizedKey[] calldata newKeys
) external returns (address newAccount) {

    // ① Verify creation proof → creation_nullifier
    uint256 creationNullifier = _verifyCreationProof(creationProof);

    // ② Read current generation
    uint256 generation = accountCreations[creationNullifier];
    require(generation > 0);

    // ③ Verify new account proof (generation N)
    //   action  = keccak256(abi.encodePacked(WORLD_CHAIN_RP_ID, uint256(0), generation))
    //   signal  = keccak256(abi.encode(newKeys)) >> 8
    uint256 newAccountNullifier = _verifyAccountProof(generation, newKeys, accountProof);

    // ④ Verify destruction proof (generation N-1)
    //   action  = keccak256(abi.encodePacked(WORLD_CHAIN_RP_ID, uint256(0), generation - 1))
    //   signal  = keccak256(abi.encodePacked("DESTROY", newAccountNullifier)) >> 8
    //   → proves same leaf_index owns old account
    //   → signal links old account to new account
    _verifyDestructionProof(destructionProof, oldAccountNullifier, newAccountNullifier, generation - 1);

    // ⑤ Nullifier uniqueness
    require(!spentAccountNullifiers[newAccountNullifier]);
    spentAccountNullifiers[newAccountNullifier] = true;

    // ⑥ Destroy old account
    address oldAccount = address(bytes20(keccak256(abi.encodePacked(oldAccountNullifier))));
    require(!destroyed[oldAccount]);
    destroyed[oldAccount] = true;
    //   → clear authorized keys on oldAccount

    // ⑦ Derive new address
    newAccount = address(bytes20(keccak256(abi.encodePacked(newAccountNullifier))));

    // ⑧ Transfer balance: oldAccount → newAccount

    // ⑨ Initialize new account
    //   → write newKeys, generation, EIP-7702 delegation

    // ⑩ Increment generation
    accountCreations[creationNullifier] = generation + 1;

    emit AccountRecycled(oldAccount, newAccount, generation);
}
```

### Key Delegation

The account proof's `signal_hash` commits to the authorized key set:

```
signal_hash = keccak256(abi.encode(authorized_keys)) >> 8
```

Keys are **independent** of WorldIDRegistry authenticator keys (BabyJubJub). Supported types:

| Type | ID | Key Data | Size |
|------|----|----------|------|
| Secp256k1 | `0x00` | compressed pubkey | 33 bytes |
| P256 | `0x01` | `qx \|\| qy` | 64 bytes |
| WebAuthn | `0x02` | `qx \|\| qy` | 64 bytes |

Keys stored via EIP-7702 delegation to `WORLD_ID_ACCOUNT_DELEGATE` at:

```
AUTHORIZED_KEYS_SLOT  = keccak256("worldchain.world_id_account.keys")
GENERATION_SLOT       = keccak256("worldchain.world_id_account.generation")
```

### Transaction Type `0x6f`

#### Envelope

Following EIP-2718 convention: raw type byte prefix, single RLP list, signature fields last.

```
0x6f || rlp([
    chain_id,                    // 0
    nonce,                       // 1
    max_priority_fee_per_gas,    // 2
    max_fee_per_gas,             // 3
    gas_limit,                   // 4
    to,                          // 5
    value,                       // 6
    data,                        // 7
    access_list,                 // 8
    authorization_list,          // 9
    account_nullifier,           // 10
    signature_type,              // 11  ← signature
    signature_payload,           // 12
])
```

| # | Field | Type | Description |
|---|-------|------|-------------|
| 0 | `chain_id` | `uint64` | EIP-155 chain ID |
| 1 | `nonce` | `uint64` | Account nonce |
| 2 | `max_priority_fee_per_gas` | `uint256` | EIP-1559 priority fee |
| 3 | `max_fee_per_gas` | `uint256` | EIP-1559 max fee |
| 4 | `gas_limit` | `uint64` | Gas limit |
| 5 | `to` | `address \| empty` | Call target (empty = CREATE) |
| 6 | `value` | `uint256` | Wei value |
| 7 | `data` | `bytes` | Call data |
| 8 | `access_list` | `[[addr, [key...]]]` | EIP-2930 |
| 9 | `authorization_list` | `[[chain_id, addr, nonce, v, r, s], ...]` | EIP-7702 |
| 10 | `account_nullifier` | `uint256` | Derives sender address |
| 11 | `signature_type` | `uint8` | `0x00` / `0x01` / `0x02` |
| 12 | `signature_payload` | `bytes` | Type-specific signature |

#### Sender Derivation

```
sender = address(bytes20(keccak256(abi.encodePacked(account_nullifier))))
```

#### Signing Hash

Type byte included. Signature fields excluded (fields 0–10 only):

```
signing_hash = keccak256(0x6f || rlp(fields[0..=10]))
```

Signed message binds to the account identity:

```
message = keccak256(signing_hash || account_nullifier)
```

#### Signature Payload by Type

| `signature_type` | `signature_payload` |
|-------------------|---------------------|
| `0x00` Secp256k1 | `rlp([y_parity, r, s])` |
| `0x01` P256 | `rlp([r, s])` |
| `0x02` WebAuthn | `rlp([authenticator_data, client_data_json, r, s])` |

#### Transaction Hash

```
tx_hash = keccak256(0x6f || rlp(fields[0..=12]))
```

#### Receipt

```
0x6f || rlp([status, cumulative_gas_used, logs_bloom, logs])
```
